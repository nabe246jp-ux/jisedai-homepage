次世代ホームページ — GitHub アップロード手順

非エンジニアの方でも 順番にコピー&ペースト するだけで GitHub に置ける ように 細かく書いています。

0準備するもの

  • GitHub のアカウント (お持ちです)
  • GitHub の「Personal Access Token (PAT)」 (以前 設定したものを 使います)
  • Windows の PowerShell (スタートメニュー → 「PowerShell」と入力 → クリック)
Personal Access Token をどこかに 控えていなければ、GitHub にログインして 右上の自分のアイコン → Settings → Developer settings → Personal access tokens から 新しく 発行 してください。発行 時 「repo」と「workflow」の チェック を 入れる の が ポイントです。

1古い .git フォルダ を 削除する

ぼくが 途中まで 用意 した 古い .git フォルダが 残っているので、まず 削除します。PowerShell を 起動 して、下のコマンド を そのまま 貼り付けて Enter してください。 
cd 'C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ'; if (Test-Path .git) { Remove-Item -Recurse -Force .git }
.git フォルダ が 元々 無ければ 何も 起きません。エラー も 出ません。安全 な コマンド です。

2git の 初期設定 (初回のみ)

git に 「自分は こういう 名前 と メールアドレス です」と 教えます。すでに 設定済みでも 上書き で 問題ない の で 実行 OK です。 
git config --global user.name 'nabe'; git config --global user.email 'nabe246jp@gmail.com'

3git を 初期化 して、ファイルを 全部 登録 する

下の コマンド を そのまま 貼り付けて Enter。プロジェクト の 全ファイルを git に 登録 して、最初の コミット を 作ります。 
git init -b main; git add .; git commit -m 'first commit: 次世代ホームページ 3つのモック'
最後に 「[main (root-commit) ...] first commit: ...」と 出れば 成功 です。

4GitHub で 空っぽ の リポジトリ を 作る

ブラウザで 下記の URL を 開いて、新しい リポジトリ を 作ります。
https://github.com/new
入力する内容:
jisedai-homepage
次世代ホームページ プロジェクト (任意)
Public (公開) または Private (非公開) どちらでも OK
重要: 下の「Initialize this repository with a README」「.gitignore」「license」の 3つの チェックは すべて 外したまま(=何も 追加 しない)で 「Create repository」を 押してください。すでに ローカル に README が ある ので、追加 すると ぶつかります。

5作った リポジトリ の URL を コピー

リポジトリ作成 後、画面 に「Quick setup」と 書かれた 場所 が 出ます。HTTPS タブ の URL を コピー します。だいたい こんな 形 です:
https://github.com/あなたのGitHubユーザー名/jisedai-homepage.git

6ローカル と GitHub を 繋ぐ

下の コマンド の 「YOUR-USERNAME」を 自分の GitHub ユーザー名 に 書き換えて 実行 します。 
git remote add origin https://github.com/YOUR-USERNAME/jisedai-homepage.git
「YOUR-USERNAME」は そのまま では 動きません。必ず 自分の GitHub ID に 置き換えてください。

7GitHub に アップロード する

下の コマンド を 実行 して、ファイル を GitHub に 送信 します。 
git push -u origin main
初回 だけ、ユーザー名 と パスワード を 聞かれます。
  • Username: GitHub ID
  • Password: Personal Access Token (発行 して 控えていた もの。普通の パスワード では ありません)
最後 に 「main -> main」と 表示 されたら、アップロード 完了 です。

8確認

ブラウザで 下記 URL を 開いて、ファイル が 並んで いれば 成功 です。
https://github.com/あなたのGitHubユーザー名/jisedai-homepage
ついで に GitHub Pages を 設定 すると、世界中 から ブラウザ で 直接 街バージョン を 見られる ように なります。Settings → Pages から、Source を「main」「(root)」に 設定 して 保存 すれば、数分後 に 公開 URL が 出ます。

★うまく いかなかった とき

  • 「git: command not found」 → Git が インストール されていません。Git for Windows を ダウンロード して インストール してください。
  • 「Authentication failed」 → パスワード の 場所 に Personal Access Token を 入れて ください。GitHub の 普通の パスワード では 通りません。
  • 「remote origin already exists」 → コマンド git remote remove origin を 実行 して から ステップ 6 を やり直して ください。
  • 「nothing to commit」 → ステップ 3 で すでに コミット 済み です。ステップ 4 へ 進んで OK。
