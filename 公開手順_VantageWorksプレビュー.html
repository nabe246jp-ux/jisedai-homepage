Deploy Guide

VantageWorks プレビューを Web に公開する。

既存の Vercel デプロイ（smart-site-mvp.vercel.app）に、今回のプレビューHTMLを追加で乗せる形にします。PowerShellで3つのコマンドを順番に貼り付ければ、1〜2分後に公開URLからアクセスできます。

公開後のURL： https://smart-site-mvp.vercel.app/vantageworks.html
このURLをそのまま他の人に送れば、PCでもスマホでも見られます。

1. プロジェクトフォルダへ移動

PowerShell を開いて、コピーで貼り付け

Windowsキーを押して「powershell」と検索 → PowerShell を起動。下のコマンドを貼り付けてEnter。

cd 'C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ\smart-site-mvp'

2. プレビューHTMLを公開フォルダにコピー

同じPowerShellで続けてコピーを実行

下のコマンドを貼り付けてEnter。public フォルダがなければ自動で作られます。

New-Item -ItemType Directory -Force -Path .\public | Out-Null
Copy-Item '..\VantageWorks_SmartSite_preview.html' '.\public\vantageworks.html' -Force

3. GitHub へ送信して、Vercelに自動デプロイ

git で送信

下のコマンドを順番に貼り付け。Vercelに繋がっていれば、自動で公開ビルドが走ります。

git add public/vantageworks.html
git commit -m "publish: vantageworks smart site preview"
git push origin main

git push の途中で「Username for GitHub」「Password」を求められたら、いつもどおりGitHubのユーザー名とパスワード（または個人アクセストークン）を入力してください。

4. 1〜2分待って、URLを開く

Vercel の自動デプロイ完了を待つ

git push 後、Vercel が自動でビルド＆公開します。所要時間は 1〜2分です。完了したら下のURLが他の人にも見られるようになります。

公開URL
https://smart-site-mvp.vercel.app/vantageworks.html
 →

つまづいたとき

「git push したのに反映されない」ときは、Vercelのダッシュボード（vercel.com）にログインして、smart-site-mvp プロジェクトの Deployments タブを確認すると、ビルド状況がわかります。

「URL を開いたら 404」になったら、デプロイがまだ走っている途中の可能性があります。30秒〜2分ほど待って再読み込みしてみてください。

後でプレビューを更新したくなったら

同じ手順 2〜3 を繰り返せば、上書きで反映されます。プレビューHTML（VantageWorks_SmartSite_preview.html）を私が更新したあと、再度 PowerShell でコピー → git push の流れです。
