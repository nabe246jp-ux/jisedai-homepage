全プロジェクト 自動同期 セットアップ手順

Documents\Claude\Projects と Documents\Claude\Artifacts の 配下 に ある すべての GitHub プロジェクト を、保存 する だけ で 自動 で 公開 ページ に 反映 させます。

でき上がる 状態

1. 編集どの プロジェクト で 何 を 編集しても
→
2. 自動 push30秒 以内 に GitHub へ 自動 反映
→
3. 公開 ページ1〜2分 後 に 自動 更新
監視 対象 (この フォルダ の 中 に ある GitHub プロジェクト 全部)
C:\Users\nabe2\Documents\Claude\Projects\
C:\Users\nabe2\Documents\Claude\Artifacts\

A の 「共通 の 準備」 は 全 プロジェクト で 一度 だけ。B の 「プロジェクト ごと の 接続」 は その プロジェクト で 一度 だけ。あとは 自動 で 動き 続けます。

A. 共通 の 準備 (生涯 一度 だけ)

自動同期 の 仕掛け を 作る

1

Git が 入って いるか 確認

PowerShell を 起動 (スタート メニュー で 「PowerShell」 と 入力 → クリック)。下の コマンド を 貼り付けて Enter。
git --version
バージョン 番号 (例: git version 2.45.0) が 出れば OK。
「認識 されていません」 などの エラー が 出たら、まず Git for Windows を ダウンロード ・ インストール。設定 は 全部 デフォルト で OK。終わったら PowerShell を 一度 閉じて 開き直し、もう 一度 上 の コマンド を 試して ください。
2

名前 と メアド を 登録 (初回 のみ)

既に 設定 済み なら 上書き で 問題 ない ので 実行 OK。
git config --global user.name 'nabe'; git config --global user.email 'nabe246jp@gmail.com'
3

自動同期 スクリプト の 置き場所 を 作る

自動同期 スクリプト 専用 の フォルダ を 作ります。
New-Item -ItemType Directory -Path 'C:\Users\nabe2\AppData\Local\auto-sync' -Force | Out-Null
4

自動同期 スクリプト を 作成

下の 長い コマンド を 一気 に コピー して、PowerShell に 貼り付けて Enter。スクリプト 本体 が 作られます。
@'
$ErrorActionPreference = 'Continue'
$watchFolders = @(
    'C:\Users\nabe2\Documents\Claude\Projects',
    'C:\Users\nabe2\Documents\Claude\Artifacts'
)
$logFile = 'C:\Users\nabe2\AppData\Local\auto-sync\auto-sync.log'
function Write-Log($msg) {
    $time = Get-Date -Format 'yyyy-MM-dd HH:mm:ss'
    "$time  $msg" | Out-File -FilePath $logFile -Append -Encoding UTF8
}
Write-Log 'auto-sync started'
while ($true) {
    Start-Sleep -Seconds 30
    foreach ($watchRoot in $watchFolders) {
        if (-not (Test-Path $watchRoot)) { continue }
        $repos = Get-ChildItem -Path $watchRoot -Directory -ErrorAction SilentlyContinue | Where-Object { Test-Path (Join-Path $_.FullName '.git') }
        foreach ($repo in $repos) {
            $repoPath = $repo.FullName
            if (Test-Path (Join-Path $repoPath '.no-auto-sync')) { continue }
            try {
                Set-Location $repoPath
                $remote = git remote get-url origin 2>$null
                if (-not $remote) { continue }
                $status = git status --porcelain 2>$null
                if ($status) {
                    Write-Log "[$($repo.Name)] changes detected"
                    git add . 2>&1 | Out-Null
                    git commit -m "auto-sync $(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')" 2>&1 | Out-Null
                    git pull --rebase 2>&1 | Out-Null
                    $pushResult = git push 2>&1
                    Write-Log "[$($repo.Name)] pushed: $pushResult"
                }
            } catch {
                Write-Log "[$($repo.Name)] error: $_"
            }
        }
    }
}
'@ | Out-File -FilePath 'C:\Users\nabe2\AppData\Local\auto-sync\auto-sync.ps1' -Encoding UTF8
これ で C:\Users\nabe2\AppData\Local\auto-sync\auto-sync.ps1 が 出来上がります。
5

テスト 起動

下の コマンド で テスト 起動。新しい 黒い PowerShell ウィンドウ が 開いて、ずっと 動き 続けます。
Start-Process powershell -ArgumentList '-ExecutionPolicy Bypass -File C:\Users\nabe2\AppData\Local\auto-sync\auto-sync.ps1'
この 黒い ウィンドウ が 「自動同期 さん」。閉じる と 同期 が 止まります。今 は 開いた まま に して おいて ください。
6

パソコン 起動時 に 自動 起動

起動 する たび に 自動 で 動く ように、Windows の スタートアップ に 登録 します。
$WshShell = New-Object -ComObject WScript.Shell; $shortcut = $WshShell.CreateShortcut([System.IO.Path]::Combine([Environment]::GetFolderPath('Startup'), 'auto-sync.lnk')); $shortcut.TargetPath = 'powershell.exe'; $shortcut.Arguments = '-WindowStyle Hidden -ExecutionPolicy Bypass -File C:\Users\nabe2\AppData\Local\auto-sync\auto-sync.ps1'; $shortcut.WorkingDirectory = 'C:\Users\nabe2\AppData\Local\auto-sync'; $shortcut.Save()
完了。これで パソコン を 再起動 する たび に、自動 で バックグラウンド で 動き始めます (画面 に は 何も 出ません)。
B. プロジェクト ごと の 接続 (その プロジェクト で 一度 だけ)

各 フォルダ を GitHub と 結ぶ

対象 プロジェクト ごと に 一度 だけ 行う 作業 です。終わった プロジェクト は、もう 二度と やる 必要 が ありません。
パターン は 2つ あります。

パターン A新規 プロジェクト (GitHub に まだ 何 も ない)
パターン BGitHub に 既に ファイル が ある (例: 次世代ホームページ)
A

パターン A: 新規 プロジェクト

パソコン に は ファイル が ある が、GitHub に は まだ 何 も 置いて いない 場合。
A-1. GitHub に 空 の リポジトリ を 作る
ブラウザ で github.com/new を 開き、Repository name を 入力 して 「Create repository」。
「README」「.gitignore」「license」 の チェック は 必ず 全部 外す (空 の まま 作る)。
A-2. 接続 コマンド (4行 まとめて)
下 の コマンド の 2 箇所 を 自分 用 に 書き換えて から 実行。
「[ファイルの場所]」 を プロジェクト フォルダ の 場所 に。
「[GITHUB_URL]」 を 作った リポジトリ の URL (.git 付き) に。
cd '[ファイルの場所]'; git init -b main; git add .; git commit -m 'first commit'; git remote add origin [GITHUB_URL]; git push -u origin main
記入例:
[ファイルの場所] → C:\Users\nabe2\Documents\Claude\Projects\my-new-app
[GITHUB_URL] → https://github.com/nabe246jp-ux/my-new-app.git
初回 だけ ブラウザ 認証 が 求められ ます。Authorize ボタン を 押す だけ。次回 から は 何 も 聞かれ ません。
B

パターン B: GitHub に 既に ファイル が ある

GitHub に は ファイル が 上がって いる が、パソコン 側 が まだ git で 接続 されて いない 場合 (例: 次世代ホームページ)。
B-1. 接続 コマンド (5行 まとめて)
下 の コマンド の 2 箇所 を 書き換えて から 実行。
cd '[ファイルの場所]'; git init -b main; git remote add origin [GITHUB_URL]; git fetch origin; git reset --soft origin/main; git add .; git commit --amend --no-edit; git push -u origin main
「次世代ホームページ」 の 場合 の 記入例:
[ファイルの場所] → C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ
[GITHUB_URL] → https://github.com/nabe246jp-ux/jisedai-homepage.git
!

「次世代ホームページ」 を 今 すぐ 接続 する 場合

パターン B の 記入例 を 使った コマンド を そのまま 貼って 実行 すれば、すぐ 自動同期 対象 に 入ります。
cd 'C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ'; git init -b main; git remote add origin https://github.com/nabe246jp-ux/jisedai-homepage.git; git fetch origin; git reset --soft origin/main; git add .; git commit --amend --no-edit; git push -u origin main
C. 確認 と 止め方

動いて いるか / 止め たい とき

1

ログ で 動作 確認

いつ どの プロジェクト が 同期 された か、ログ で 見られ ます。
Get-Content 'C:\Users\nabe2\AppData\Local\auto-sync\auto-sync.log' -Tail 30
「[プロジェクト名] changes detected」 → 変更 が 見つかった
「[プロジェクト名] pushed: ...」 → GitHub に 送信 完了
2

一時 停止

タスク マネージャー (Ctrl + Shift + Esc) で 「Windows PowerShell」 を 選んで 「タスクの終了」。次に パソコン を 起動 すれば また 動き出し ます。
3

特定 の プロジェクト だけ 同期 を 止め たい

その プロジェクト フォルダ に 「.no-auto-sync」 と いう 空 の ファイル を 作る だけ。それ が ある 限り、その プロジェクト は 同期 されません。
コマンド 例:
New-Item -Path '[プロジェクトフォルダ]\.no-auto-sync' -ItemType File -Force | Out-Null
4

完全 に 止める (二度と 動か さない)

スタートアップ から 削除 します。
Remove-Item ([System.IO.Path]::Combine([Environment]::GetFolderPath('Startup'), 'auto-sync.lnk')) -Force
★

うまく いかない 時

  • 「git: command not found」 → Git for Windows が 入って いません。ステップ 1 の リンク から インストール。
  • 初回 push で 「Authentication failed」 → Personal Access Token が 必要 な モード。GitHub の Settings → Developer settings → Personal access tokens で 新規 発行 (権限 「repo」)。Username に GitHub ID、Password に PAT を 入れる。
  • 同期 が 動かない → ステップ C-1 の ログ を 確認。何 も ログ が 出て いない なら、自動同期 さん が 起動 して いない 可能性。タスク マネージャー で 「Windows PowerShell」 が 動いて いるか 確認。動いて いない なら、ステップ A-5 の テスト 起動 を もう 一度 実行。
  • ログ に 「rejected」 と 出る → ブラウザ で GitHub を 直接 編集 した 履歴 と ぶつかった 可能性。下 の コマンド で 解消:
    cd '[該当 プロジェクト フォルダ]'; git pull --rebase origin main
  • ログ に 「fatal: ambiguous argument 'origin/main'」 → パターン B で main 以外 の ブランチ 名 (master 等) の 場合。下 の コマンド で 確認:
    git branch -r で remote の ブランチ 名 を 見て、 origin/main の 部分 を 置き換える。
全部 終わったら、以後 は ファイル を 普通 に 保存 する だけ で、全プロジェクト が 自動 で GitHub に 反映 されます。
