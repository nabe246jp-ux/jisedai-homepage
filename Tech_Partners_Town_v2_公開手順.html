Tech Partners Town v2 公開手順

他の人にURLで見せられるように公開する手順です。
Vercel（最速・推奨）と GitHub Pages（既存サイトと共存）の両方を案内します。

このルートで得られるもの

所要約5分。一度設定すれば、以降はGitHubに何かpushするたびに自動でビルド＆公開されます。Next.jsとの相性が最高で、Three.jsの3D表現も含めて何もいじらずに動きます。

1

Vercel にサインアップする

下のサイトを開いて「Continue with GitHub」をクリックします。

https://vercel.com/signup
既にVercelアカウントを持っている場合は、ログインして次へ進んでください。
2

GitHubアカウントとの連携を許可する

「Authorize Vercel」のボタンを押して、Vercelに自分のGitHubリポジトリを見る権限を与えます。
続いて「Install Vercel」のボタンが出るので、All repositories（全リポジトリ）か、jisedai-homepage だけを選んで「Install」を押します。

どちらでも今回の作業は同じです。迷ったら「jisedai-homepage」だけ選ぶ方が安全です。
3

新しいプロジェクトを作る

ダッシュボード右上の「Add New...」→「Project」を押します。
リポジトリ一覧の中から「jisedai-homepage」を見つけて、その横の「Import」ボタンを押します。

4

大事な設定：Root Directory を変える

設定画面が出たら、「Root Directory」の項目で「Edit」を押し、下の値を入力してください。

tech-partners-town-v2

これでVercelは「リポジトリ全体ではなく、tech-partners-town-v2のフォルダだけがNext.jsプロジェクト」と理解します。

ここを忘れるとビルドが必ず失敗します。かならずRoot Directoryを「tech-partners-town-v2」に設定してから次へ進んでください。
5

プロジェクト名を設定する

「Project Name」の項目に下の名前を入れます。

tech-partners-town
この名前がそのままURLになります。例：tech-partners-town.vercel.app
すでに他の人が同じ名前を使っていると、別の名前を提案されます。その時は提案に従ってください。
6

デプロイを開始する

その他の設定はそのままで、画面下の「Deploy」ボタンを押します。

1〜3分で「Congratulations!」の画面と一緒に町のスクリーンショットが表示されたら成功です。

7

公開URLを取得する

完了画面の中央に表示される下のような形式のURLが、公開URLです。

https://tech-partners-town.vercel.app

このURLを誰かに送れば、その人もブラウザで町を見られます。

スマホからも開けます。ただし3D描画なので、5年以上前の機種では動きが重い可能性があります（推測です）。
8

これ以降のアップデート方法

もう何もしなくて大丈夫です。Claude Codeからプロジェクトに変更を加えてGitHubに反映されると、Vercelが自動で検知してビルド・更新します。
通常2〜3分で新バージョンが公開URLに反映されます。

このルートで得られるもの

既存のGitHub Pagesにv2を追加します。モック一覧（v1〜v3）はそのまま残り、新しくv2が追加されます。GitHub以外のサービスを使わずに完結させたい場合に。

1

変更をGitHubに反映する（自動同期に任せる）

すでに30秒ごとの自動同期が動いているので、特別な操作は不要です。
追加された「.github/workflows/deploy-pages.yml」と「next.config.mjs」の更新が自動でGitHubに上がります。
急ぎたい場合は、Claude Code で下を実行してください。

cd "C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ"
git add -A
git commit -m "v2: GitHub Pages対応とサブパス公開"
git push
2

GitHubのリポジトリ画面を開く

https://github.com/nabe246jp-ux/jisedai-homepage
3

Pagesの公開元を「GitHub Actions」に切り替える

リポジトリ画面の上部メニューから「Settings」をクリック。
左サイドバーの「Pages」を選択。
「Build and deployment」の「Source」項目を「Deploy from a branch」から「GitHub Actions」に変更。

ここがポイント：これまで「mainブランチから直接配信」だった場合、新しく追加したワークフローが代わりに公開を担います。既存のv1〜v3モックもワークフロー内でコピーされるので消えません。
4

初回ビルドを開始する

リポジトリ画面の上部メニューから「Actions」タブをクリック。
左サイドバーの「Deploy GitHub Pages」を選択。
画面右側の「Run workflow」を押し、表示された「Run workflow」ボタンをもう一度押します。

2〜5分でビルドが完了します。緑のチェックマークが付けば成功。赤い×が出たらクリックして詳細を見て、エラー文をClaudeに貼り付けてください。
5

公開URLにアクセスする

下の2つのURLが公開されます。

https://nabe246jp-ux.github.io/jisedai-homepage/

↑ これまで通りのモック一覧（v2のカードが新しく追加されている）

https://nabe246jp-ux.github.io/jisedai-homepage/tech-partners-town-v2/

↑ v2の3Dの町

モック一覧の「TECH PARTNERS TOWN v2」のカードからもジャンプできます。
6

これ以降のアップデート方法

Vercelと同じく、GitHubに変更が反映されれば自動でビルド・公開が走ります。
Actionsタブで進捗を確認できます。
