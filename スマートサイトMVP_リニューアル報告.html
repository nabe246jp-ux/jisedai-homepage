Live Site OS · Smart Site MVP

スマートサイトMVPを、
高級感ある自社プロダクトとして刷新しました。

Next.jsプロジェクトをGitHubから取得し、デザインを黒基調＋シアン×紫×ピンクのオーロラ・グラデへ全面刷新。コピーは「動かないHPを、動くプロダクトへ」という、買い手の現状に切り込む方向に書き換え、Header/Footer/Aurora背景を新規分離してコンポーネント設計も整えました。タイプチェック（tsc --noEmit）はエラーゼロで通過済みです。

変更後プロジェクト一式を開く GitHub リポジトリ 公開URL（更新前）

1. 何を変えたか（要点）

デザインの方向性

ライトな白背景＋teal/coralのSaaSトーンから、深い宇宙黒（#07090F）＋シアン/紫/ピンクのオーロラ・グラデへ刷新。背景にはオーロラ光球とグリッド模様を重ね、ガラスモーフィズム調の半透明カードと極細グラデ・ボーダーで奥行きを出しました。タイポグラフィはInter＋Noto Sans JPの2軸で統一し、見出しは大きく太く、字間を詰めてプロダクト感を強くしています。

ブランド軸

プロダクト名「Smart Site / スマートサイト」とサブブランド「Live Site OS」の二段ロックアップを、ヘッダーとフッターで統一。ロゴは黒地にグラデの輪郭が浮かぶマークに更新しました。

コピーの再構築

ヒーローを「動かないHPを、動くプロダクトへ。」に変更し、買い手の現状を一行で否定して動かす導入に。CTAは「体験を開始」→「デモを体験する」、相談導線は「30分相談を予約」と行動を具体化しました。各セクションの見出しもプロダクト視点（数字で語る／案内役／4ステップで動く）に書き換えています。

コンポーネント分割

Hero内に埋まっていたサイト共通要素を分離し、固定追従型のHeader、サイト全体のFooter、ヒーローと相談セクションで再利用するAurora背景、ロゴ用BrandMarkを新規追加。framer-motionを使う全コンポーネントに「use client」を明示し、Server/Client境界を整えました。

4. 動作確認の手順（あなたのパソコンで実行）

変更ファイル一式は C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ\smart-site-mvp\ に置きました。リモート（GitHub）も設定済みなので、そのままプッシュできます。

PowerShellを開いてフォルダへ移動

cd 'C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ\smart-site-mvp'

パッケージをインストール

新しく追加したGoogleフォント関連の解決のため、念のため最新の依存関係を入れ直します。

npm install

ローカルで開発サーバーを起動して見た目を確認

npm run dev

ブラウザで http://localhost:3000 を開いて確認してください。Ctrl + C で終了します。

本番ビルドが通るかを確認

npm run build

このコマンドはVercelが自動で実行するものと同じです。ローカルで通れば、Vercelのデプロイも基本的に成功します。

変更をGitHubへプッシュしてVercelに自動デプロイ

git add -A
git commit -m "feat: redesign to dark + gradient brand, add header/footer, refactor copy"
git push origin main

Vercelに連携済みなら、push後1〜2分で smart-site-mvp.vercel.app が新デザインに切り替わります。

5. 元に戻したいとき

もし全体を以前の状態に戻したい場合、以下のコマンドで 4490d00（更新前のコミット）に戻せます。

git reset --hard 4490d00
ローカルの未コミット変更は消えます。一部だけ戻したい場合は git checkout 4490d00 -- 対象ファイル で個別に戻せます。

7. 推奨される次のステップ

短期で効くもの

  • OGP画像（1200×630）を作成してSNSシェア時の見栄えを揃える
  • ヒアリングフォームをHubSpotやTallyで埋め込み、メール直リンクから一段アップグレード
  • 診断結果を「相談を予約」CTAと自動連結（結果を初期メッセージに渡す）

中期で効くもの

  • Smart Topicsを実データ（Supabase or Googleスプレッドシート）と接続
  • AIナビをClaude APIに差し替え、サイト内ドキュメントをコンテキストとして回答
  • 計測（Vercel Analytics + Plausible / GA4）で訪問者タイプ別の離脱を可視化

長期の世界観

「Live Site OS」というメタブランドの世界観をプロダクトサイトと自社コーポレートサイトで二重に持たせ、自社サイトを「最高の事例」として運用するのが効果的です。次世代ホームページ プロジェクトの街バージョン（Tech Partners Town）と組み合わせて、ショーケースの厚みを作っていけます（推測です）。
