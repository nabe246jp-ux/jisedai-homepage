自動同期 セットアップ手順

パソコンのフォルダで保存すると、30秒以内に GitHub と 公開ページが 自動更新される 仕組みを 作ります。

これが できるように なります

1. パソコンファイルを 保存
→
2. GitHub30秒以内に 自動で 反映
→
3. 公開ページ1〜2分後に 自動で 更新

セットアップは一度だけ。あとは パソコンを 起動 したら 勝手に 動き続けます。

A. 準備 (一度だけ)

パソコンと GitHub を つなぐ

1

Git が 入っているか チェック

PowerShell を 起動 します。スタートメニュー → 「PowerShell」と 入力 → クリック。下の コマンド を 貼り付けて Enter。
git --version
バージョン番号が表示されたら OK (例: git version 2.45.0)
「git: command not found」「'git' は…認識されていません」と出たら、まず Git for Windows を ダウンロード して インストール して ください。インストール時の 設定は すべて デフォルト で OK。終わったら PowerShell を 一度 閉じて 開き直し、もう一度 上の コマンド を 試してください。
2

フォルダに 移動

プロジェクトフォルダに 移動 します。
cd 'C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ'
3

名前と メアドを 登録 (初回のみ)

既に 登録済み なら 上書きされる だけ なので 安全です。
git config --global user.name 'nabe'; git config --global user.email 'nabe246jp@gmail.com'
4

フォルダを GitHub と 接続

下の3つの コマンドを 1つずつ 順番に 実行 してください。
git init -b main
git remote add origin https://github.com/nabe246jp-ux/jisedai-homepage.git
「remote origin already exists」と 出たら、下の コマンドで 上書き してください。
git remote set-url origin https://github.com/nabe246jp-ux/jisedai-homepage.git
5

GitHub の 中身を ローカルに 取り込む

GitHub に 既に アップロード 済みの ファイルを ローカルの git 履歴 に 取り込みます。
git fetch origin; git reset --soft origin/main; git add .; git commit --amend -m 'sync local with github' --no-edit 2>$null; git status
これで パソコンの ファイルが 「GitHub の 続き」と して 認識される ように なります。
6

初回の push (認証情報を 保存)

下の コマンドを 実行。初回 だけ ブラウザ で 認証 (GitHub に サインイン している はず なので、ボタンを 押す だけ) が 求められます。
git push -u origin main
「Sign in with your browser」と 出たら、開いた ブラウザで Authorize ボタン を 押すと 認証 完了。次回からは 何も 聞かれません。

ブラウザが 開かない 場合は、ユーザー名 と パスワード が 聞かれます。Username に GitHub ID、Password に Personal Access Token を 入れて ください。
「main -> main」と 表示 されたら、これで パソコンと GitHub は 完全に つながりました。
B. 自動同期 (一度だけ)

30秒ごとに 自動で 送る 仕掛け

7

除外設定の 追記

自動同期スクリプトの ログ などを GitHub に 上げ ない ように、除外 設定 を 追加 します。
Add-Content -Path .gitignore -Value "`r`n# auto-sync`r`nauto-sync.log`r`n.auto-sync.log"
8

自動同期スクリプトを 作る

下の 長い コマンド を そのまま 貼り付けて Enter。フォルダ内に 「auto-sync.ps1」と いう ファイル が 作られます。
@"
`$ErrorActionPreference = 'Continue'
`$folder = 'C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ'
`$logFile = Join-Path `$folder '.auto-sync.log'
function Write-Log(`$msg) {
    `$time = Get-Date -Format 'yyyy-MM-dd HH:mm:ss'
    "`$time  `$msg" | Out-File -FilePath `$logFile -Append -Encoding UTF8
}
Set-Location `$folder
Write-Log "auto-sync started"
while (`$true) {
    Start-Sleep -Seconds 30
    try {
        Set-Location `$folder
        `$status = git status --porcelain 2>`$null
        if (`$status) {
            Write-Log "changes detected"
            git add . 2>&1 | Out-Null
            git commit -m "auto-sync `$(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')" 2>&1 | Out-Null
            git pull --rebase origin main 2>&1 | Out-Null
            `$pushResult = git push origin main 2>&1
            Write-Log "pushed: `$pushResult"
        }
    } catch {
        Write-Log "error: `$_"
    }
}
"@ | Out-File -FilePath 'auto-sync.ps1' -Encoding UTF8
これで プロジェクト フォルダ の 中 に auto-sync.ps1 が 出来ます。
9

テスト 起動 (動くか 確認)

下の コマンドで テスト 起動 します。新しい 黒い PowerShell の ウィンドウが 開いて、ずっと 動き続けます。
Start-Process powershell -ArgumentList '-ExecutionPolicy Bypass -File auto-sync.ps1'
この 黒い ウィンドウ が 「自動同期さん」 です。閉じる と 同期 が 止まります。
テストとして、何か 1つ ファイル を 編集 して 保存。30秒 〜 1分 待つ と、GitHub の リポジトリ ページに 自動で 反映 されている はずです。
https://github.com/nabe246jp-ux/jisedai-homepage
10

パソコン起動時 に 自動で 動かす

毎回 手動で 起動 するのは 面倒 なので、Windows が 起動 したら 勝手に 動く ように 設定 します。
まず、Windows の スタートアップ フォルダ を 開きます。
explorer shell:startup
エクスプローラ が 開きます。そこに、ショートカット を 作る ための コマンド を 続けて 実行 します。
$WshShell = New-Object -ComObject WScript.Shell; $shortcut = $WshShell.CreateShortcut([System.IO.Path]::Combine([Environment]::GetFolderPath('Startup'), 'jisedai-auto-sync.lnk')); $shortcut.TargetPath = 'powershell.exe'; $shortcut.Arguments = '-WindowStyle Hidden -ExecutionPolicy Bypass -File "C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ\auto-sync.ps1"'; $shortcut.WorkingDirectory = 'C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ'; $shortcut.Save()
これで パソコン を 再起動 する たび に、自動で 「自動同期さん」が バックグラウンド で 動き始めます。画面 には 何も 出ません。
C. 確認 と 止め方

動いて いるか / 止め たい とき

11

動いて いるか 確認

「ログ ファイル」 を 見れば、いつ 同期 が 走った か わかります。
Get-Content 'C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ\.auto-sync.log' -Tail 20
「タスク マネージャー」(Ctrl + Shift + Esc) で 「Windows PowerShell」 が 動いて いれば、自動同期 さん が 起きて います。
12

止めたい 時

タスク マネージャー で 「Windows PowerShell」 を 選んで 「タスクの終了」 を 押すと 止まります。次に パソコン を 起動 すれば また 自動 で 動き出します。
完全 に 止めたい (起動時 も 動かさない) 場合は、スタートアップ フォルダ から ショートカット を 削除 してください。
explorer shell:startup
「jisedai-auto-sync.lnk」 を 削除 すれば 完了です。
★

うまく いかない 時

  • ステップ 5 で エラー → 「git fetch origin」 を 単独で 実行 して、エラー文 を 見せて ください。
  • ステップ 6 で 認証 失敗 → Personal Access Token が 古い 可能性。GitHub の Settings → Developer settings → Personal access tokens で 新規発行 してください (権限は 「repo」)。
  • 同期 が 走らない → ステップ 11 の ログ を 確認。「changes detected」 が 出ていない なら ファイル の 変更 が 検出 されていない (場所 を 間違えている?) かも。
  • push が 失敗 → ログに 「rejected」 と 出ている なら、ブラウザ で GitHub を 直接 編集 してしまった 可能性。下の コマンド で 一旦 GitHub の 内容 を 取り込みます。
    cd 'C:\Users\nabe2\Documents\Claude\Projects\次世代ホームページ'; git pull --rebase origin main
全部 終わったら、以後 は ファイル を 普通 に 保存 する だけ で 公開 ページ に 反映 されます。
